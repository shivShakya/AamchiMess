# Hostel Mess Management Android Application

This project was developed as part of a Software Engineering course to address real-world challenges in hostel mess management. It provides an automated solution for tracking subscriptions, meal bookings, and payments, replacing inefficient manual processes.

---

## Development Process

Adhering to standard Software Engineering best practices, the project was executed through the following stages:

* **Requirement Analysis**: Identifying pain points of students and mess administrators.
* **Design & Architecture**: Creating UI wireframes and backend schema.
* **Development**: Implementing a modular frontend and a scalable API.
* **Testing**: Ensuring reliability via Postman and unit testing.
* **Deployment**: Setting up the server environment for live interaction.

---

## Key Features

* **Student Dashboard**: User-friendly interface for managing mess subscriptions.
* **Meal Management**: Booking and cancellation system to help students plan ahead.
* **Sustainability**: Optimized booking system to significantly reduce food waste.
* **Automated Notifications**: Real-time alerts for menu updates and payment reminders.
* **Admin Panel**: Streamlined interface for mess managers to track daily orders and payments.
* **REST Integration**: Seamless data exchange between the Android app and the backend server.

## Installation & Setup

1. **Frontend**:
   * Open the `/android` folder in Android Studio.
   * Build the project and run it on an emulator or physical device.

2. **Backend**:
   * Navigate to the `/backend` directory.
   * Install dependencies: `pip install -r requirements.txt`
   * Run the server: `python app.py`

3. **API Configuration**:
   * Ensure the Android app points to the correct server IP address in the configuration file.
